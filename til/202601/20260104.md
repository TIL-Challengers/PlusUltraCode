### GOF 디자인 패턴 책 100쪽 후기

오늘은 디자인 패턴 관련된 책이 궁금하여 읽게 되었다.

관련된 책은 되게 재밌다. 그림 위주로 나와 있고 예시도 단순명료하여 이해가 쉬웠다.

오늘 배운 디자인 패턴은 전략 패턴과 옵저버 패턴 2가지 였다.

간단하게 복습겸 설명 해보겠다.

1. 전략 패턴

문제 => 48마리의 오리중 누구는 날아야 되고, 누구는 날지못하고, 날지 못하다가 갑자기 날 수 있는 오리를 구현해야 하는 비지니스적 문제가 생겼다.
모든 코드를 직접 작성하면 말이 안된다.
상속을 해볼까?? => 이것 또한 모든 코드를 직접 작성하는 꼴이다.

interface를 이용하여 추상화를 composition으로 주입하면 되지 않을까? 정답이다.

해당 책에서는 Duck 이라는 추상클래스를 이용하여 어느정도 틀을 만들었고 해당 추상클래스 멤버필드를 interface로 composition선언을 해놨다.
해당 추상클래스를 상속받게 되는 여러 duck들은 자기 능력에 맞는 interface의 구현체들을 외부 클라이언트에 의해 주입받게 되면 전략 패턴 완료다.. 의외로 코드를 만들면서 짰던 기억이 많다

2. 옵저버 패턴
개념적으로 설명해 보겠다.
1: N 방식의 주제 하나가 있고 그 주제를 구독한 옵저버 들이 존재한다.
주제가 바뀌게 되면 모든 옵저버들은 해당 변경된 내용들을 인식해야 한다.
어떻게 구현할까??

interface 로 subject, observer 및 display를 만든 뒤
subject에 해당하는 클래스를 하나 만들고 implements한다. 
해당 subject는 List<Observer> observers 형태로 멤버 필드를 선언한 뒤 만약 update되는 사항들이 있으면 
아래의 코드처럼 전체 observer들을 업데이트 해준다.

여기서 interface를 List 형태로 선언했기 때문에 Observer를 구현받는 클래스들은 넣을 수 있다. 
```
for(Observer observer : observers){
  observer.update(~);
}
```

Observer에서는 update 라는 함수를 오버라이딩 해주고 update될 때마다 display를 해주면 된다.
